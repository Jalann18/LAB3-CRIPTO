// Vulnerabilidades Implementadas Intencionales

CRÍTICO

Auth Bypass (Pass-the-Hash)

El servidor acepta el hash MD5 directamente como credencial. Un atacante que capture el hash puede autenticarse sin conocer la contraseña (Reutilización de Hash).

ALTO

MD5 Sin Salt

MD5 es vulnerable a ataques de diccionario y rainbow tables. Sin salt, dos usuarios con el mismo password tienen el mismo hash.

ALTO

Hash en Cliente

El hash se genera en JavaScript (lado cliente), visible en el código fuente. Un atacante puede modificarlo antes de enviarlo.

MEDIO

User Enumeration

Mensajes de error diferentes para "usuario no existe" vs "contraseña incorrecta" permiten enumerar usuarios válidos.

MEDIO

Sin CSRF Protection

No hay tokens CSRF en los formularios. Un sitio malicioso podría hacer peticiones en nombre del usuario.

INFO

DB Leak Endpoint

El endpoint /api/users expone todos los usuarios y sus hashes sin autenticación. (Intencional para el lab)
